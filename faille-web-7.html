Révéler des informations techniques dans le HTML

Le développeur d'application web ne doit pas mettre de commentaires dans le HTML révélant :

  • Des informations comme des noms de fichiers ou des chemins d'accès
  • Des anciens et futurs liens
  • Des données sensibles
  • Des morceaux de code source

Cacher les informations sur les serveurs

La majorité des serveurs Web et des languages ont des directives pour cacher les versions dans les entêtes.

Par exemple dans Apache on peut cacher le "serveur signature".
